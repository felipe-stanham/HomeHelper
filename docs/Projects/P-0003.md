## Scope 2: Launch integration

**Branch:** `scope/P-0003-2-launch-integration`
**Status:** `[ ]`
**Depends on:** Scope 1

### Tasks
- [ ] Make `AppConfig.mcp_port` optional and not required (already `Optional[int] = None`, verify no validation enforces it)
- [ ] Update `ProcessManager.start_app()` to call `allocate_mcp_port()` for MCP-enabled apps and pass `--mcp-port` in the command
- [ ] Update `ProcessManager.stop_app()` to call `release_mcp_port()` on shutdown
- [ ] Store allocated MCP port in `MCPInfo.mcp_port` at launch time
- [ ] Update `ServiceManager.generate_service_template()` to include `--mcp-port` in `ExecStart`
- [ ] Add backward compatibility: if manifest declares `mcp_port`, log deprecation warning and use it as a hint (validate availability)
- [ ] Verify `MCPGateway` already reads from `MCPInfo.mcp_port` (no changes expected)
- [ ] Verify `HealthMonitor` MCP probe already reads from `MCPInfo.mcp_port` (no changes expected)
- [ ] Update existing unit tests for ProcessManager and ServiceManager
- [ ] Add tests to `TESTS.md` for MCP port launch integration

### Acceptance Criteria

- **cap-001:** `AppConfig.mcp_port` field is `Optional[int] = None` and not required. Apps with `mcp_server: true` and no `mcp_port` get a port allocated automatically.
- **cap-004:** When starting an MCP-enabled app, the command includes `--mcp-port <port>`. The allocated MCP port is stored in `MCPInfo.mcp_port` and in `RuntimeInfo`.
- **cap-005:** `MCPGateway` connects to the dynamically allocated port (no behavior change needed if `MCPInfo.mcp_port` is already populated correctly).
- **cap-007:** If an app declares `mcp_port: 9005`, the platform uses 9005 if available (with a deprecation warning in logs). If 9005 is taken, it allocates from the range and logs a warning.
