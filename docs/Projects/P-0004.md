# P-0004: Environment-Scoped App Services + Bootstrap Docs

**Created:** 2026-04-24 | **Status:** [DONE]

Small fix-and-document project. No `Projects/P-0004/` spec package — the scope is narrow and self-describing.

## Background

The Latarnia platform already runs as two environment-scoped system-level systemd units on the homeserver:
- `latarnia-tst.service` (TST)
- `latarnia-prd.service` (PRD)

However, **per-app** services generated at runtime by `service_manager.py` use the prefix `latarnia-` with no environment segment. Because the Pi runs both TST and PRD under the same user, installing an app with the same name in both environments would produce two `~/.config/systemd/user/latarnia-{app}.service` files that collide.

Additionally, the `latarnia-tst.service` / `latarnia-prd.service` unit files are currently hand-installed on the Pi with no documented procedure for a fresh-host bootstrap.
